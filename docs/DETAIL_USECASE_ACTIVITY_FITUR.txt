BAGIAN 1: USE CASE CREW (STAFF DEPOT)
================================================================================

CREW ADALAH: Staff/kurir depot yang bertugas membuat transaksi, mengambil pesanan,
            mengelola pembayaran, dan berinteraksi dengan pelanggan.

---

USE CASE 1.1 - LOGIN CREW
---
Nama: Login Crew
Aktor Utama: Crew (Staff Depot)
Sistem: Depo Air Mobile App
Tujuan: Staff dapat masuk ke aplikasi dengan username dan password
Precondition: Aplikasi sudah terinstall dan server running
Postcondition: Crew sudah login dan bisa akses fitur crew

Flow Utama:
1. Crew membuka aplikasi Depo Air
2. Aplikasi menampilkan halaman Login
3. Crew memilih role "CREW"
4. Crew input username
5. Crew input password
6. Crew klik tombol "Login"
7. Sistem validasi username & password ke server
8. Server mengecek di database
9. Jika benar: Server return JWT token (8 jam)
10. Sistem menyimpan token di Secure Storage
11. Sistem menyimpan data user di SharedPreferences
12. Aplikasi navigate ke Dashboard Crew
13. Use case selesai - LOGIN SUKSES

Flow Alternatif:
- Jika username/password salah: Tampilkan error message, kembali ke step 4
- Jika server down: Tampilkan error "Koneksi Server Gagal"
- Jika timeout: Tampilkan error "Request Timeout", tawarkan retry


USE CASE 1.2 - LOGOUT CREW
---
Nama: Logout Crew
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Staff keluar dari aplikasi dan session dihapus
Precondition: Crew sudah login
Postcondition: Token dihapus, kembali ke halaman login

Flow Utama:
1. Crew klik menu Settings (pada Dashboard)
2. Crew scroll dan klik tombol "Logout"
3. Sistem menampilkan dialog konfirmasi "Yakin ingin logout?"
4. Crew klik "Ya"
5. Sistem menghapus JWT token dari Secure Storage
6. Sistem menghapus user data dari SharedPreferences
7. Sistem hapus cache data lokal
8. Aplikasi navigate kembali ke Login Screen
9. Use case selesai - LOGOUT SUKSES


USE CASE 1.3 - BUAT TRANSAKSI BARU
---
Nama: Buat Transaksi Penjualan
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Crew membuat transaksi penjualan untuk pelanggan
Precondition: Crew sudah login
Postcondition: Transaksi berhasil dibuat dengan status PENDING (menunggu validasi)

Flow Utama:
1. Crew dari Dashboard klik tombol "Buat Transaksi"
2. Aplikasi menampilkan halaman "Buat Transaksi Baru"
3. Halaman menampilkan form:
   - Dropdown: Pilih Pelanggan
   - Button: Tambah Pelanggan Baru
   - Cart display (awal kosong)
   - Tombol: Tambah Produk
4. Crew memilih pelanggan dari dropdown
5. Jika pelanggan tidak ada, Crew klik "Tambah Pelanggan Baru" (lihat UC 1.8)
6. Crew klik tombol "Tambah Produk"
7. Aplikasi buka Modal: Pilih Produk
8. Modal menampilkan list semua produk dengan:
   - Nama produk, harga, stok
   - Kategori filter
   - Search bar
9. Crew scroll dan klik produk yang ingin dibeli
10. Aplikasi buka form input jumlah & harga:
    - Input field: Jumlah
    - Input field: Harga Satuan (pre-fill dari database)
    - Input field: Galon Pinjam (opsional)
    - Input field: Galon Kembali (opsional)
    - Tombol: Tambah ke Keranjang
11. Crew input jumlah (misal: 5)
12. Harga otomatis dihitung: 5 x harga satuan
13. Crew klik "Tambah ke Keranjang"
14. Item ditambah ke cart
15. Cart display update menampilkan item baru
16. Crew bisa klik "Tambah Produk Lagi" atau lanjut (repeat step 6-15)
17. Setelah selesai tambah semua item, Crew klik "Lanjut ke Pembayaran"
18. Aplikasi navigate ke halaman Pilih Metode Pembayaran (lihat UC 1.4)

Flow Alternatif:
- Jika stok tidak cukup: Tampilkan warning, tapi tetap bisa input (kuantitas > stok)
- Jika customer salah: Bisa ganti customer sebelum lanjut ke pembayaran
- Edit Item: Crew bisa klik item di cart untuk edit jumlah/harga
- Hapus Item: Crew bisa swipe/klik delete untuk hapus item dari cart


USE CASE 1.4 - PILIH METODE PEMBAYARAN
---
Nama: Pilih & Proses Metode Pembayaran
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Crew memilih metode pembayaran dan proses pembayaran
Precondition: Crew sudah memilih customer dan items
Postcondition: Pembayaran selesai, transaksi status PENDING

Flow Utama:
1. Crew di halaman Pembayaran, melihat ringkasan:
   - Nama customer
   - List items dengan total
   - Total harga transaksi
   - 3 tombol metode: CASH, QRIS, TRANSFER
2. Crew klik salah satu metode pembayaran

JIKA CASH:
3a. Aplikasi tanya nominal uang diterima
4a. Crew input nominal (misal: 100000)
5a. Sistem hitung kembalian: 100000 - total harga
6a. Tampilkan ringkasan kembalian
7a. Crew klik "Konfirmasi Pembayaran CASH"
8a. Jump ke UC 1.5 SIMPAN TRANSAKSI

JIKA QRIS:
3b. Aplikasi generate QR Code QRIS
4b. Tampilkan QR Code di screen
5b. Tampilkan pesan "Scan QR Code dengan e-wallet"
6b. Start polling status pembayaran ke server
7b. Customer scan QR Code dan bayar
8b. Aplikasi terus poll server cek status
9b. Jika pembayaran berhasil: Tampilkan ✓ "Pembayaran Diterima"
10b. Jump ke UC 1.5 SIMPAN TRANSAKSI
11b. Jika timeout (2 menit): Tampilkan "Pembayaran Expired", tawarkan retry
    - Retry: Generate QR baru, repeat step 3b
    - Cancel: Batalkan transaksi

JIKA TRANSFER:
3c. Aplikasi tampilkan: Rekening Bank Depot
4c. Crew catat/screenshot rekening
5c. Crew kasih info ke customer
6c. Customer transfer ke rekening
7c. Crew tunggu konfirmasi
8c. Bisa klik "Konfirmasi Transfer" (manual confirm)
9c. Jump ke UC 1.5 SIMPAN TRANSAKSI


USE CASE 1.5 - SIMPAN TRANSAKSI
---
Nama: Simpan Transaksi
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Transaksi disimpan ke server dan menunggu validasi manager
Precondition: Pembayaran sudah selesai
Postcondition: Transaksi BERHASIL DIBUAT, status PENDING

Flow Utama:
1. Sistem siapkan data transaksi:
   - Nomor transaksi (auto generate: TRX-20260523-001)
   - Crew ID
   - Customer ID
   - List items dengan detail
   - Total harga
   - Metode pembayaran
   - Status pembayaran
   - Tanggal/jam transaksi
   - Status: PENDING (menunggu validasi manager)
2. Sistem kirim ke server via API: POST /v1/transaksi/create
3. Server validasi data
4. Server simpan ke database
5. Server return response dengan transaksi ID
6. Sistem simpan data transaksi ke cache lokal
7. Aplikasi tampilkan dialog sukses:
   - "✓ Transaksi Berhasil Dibuat!"
   - Nomor transaksi: TRX-20260523-001
   - Total: Rp 500.000
   - Status: MENUNGGU VALIDASI MANAGER
   - 2 tombol: "Lihat Detail" dan "Kembali ke Dashboard"
8. Jika klik "Lihat Detail": Tampilkan detail lengkap transaksi
9. Jika klik "Kembali ke Dashboard": Navigate ke Dashboard Crew
10. Notifikasi dikirim ke Manager: Ada transaksi baru menunggu validasi


USE CASE 1.6 - LIHAT RIWAYAT TRANSAKSI
---
Nama: Lihat Riwayat Transaksi Crew
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Crew melihat semua transaksi yang pernah dibuat
Precondition: Crew sudah login
Postcondition: Menampilkan list riwayat transaksi

Flow Utama:
1. Crew dari Dashboard klik menu "Riwayat Transaksi"
2. Aplikasi fetch data dari server: GET /v1/transaksi?crewId=XXX
3. Aplikasi menampilkan list transaksi dengan filter:
   - Search bar: Cari berdasarkan nomor transaksi/customer
   - Filter dropdown: Semua, Pending, Diproses, Selesai, Dibatalkan
   - Sorting: Terbaru, Tertua, Tertinggi, Terendah
4. Setiap item list menampilkan:
   - Nomor transaksi (TRX-xxx)
   - Nama customer
   - Total harga
   - Status (dengan warna: pending=kuning, diproses=biru, selesai=hijau, dibatalkan=merah)
   - Tanggal
5. Crew klik 1 transaksi untuk lihat detail
6. Aplikasi menampilkan detail lengkap:
   - Nomor, tanggal, customer
   - List items dengan detail (produk, qty, harga)
   - Total harga, metode bayar
   - Status validasi (pending/disetujui/ditolak)
   - Nama manager yang validasi (jika sudah)
   - Alasan jika ditolak
7. Crew bisa scroll untuk lihat semua detail
8. Crew bisa klik "Kembali" untuk back ke list

USE CASE 1.8 - TAMBAH PELANGGAN BARU
---
Nama: Tambah Pelanggan Baru
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Crew bisa menambah data pelanggan baru ke sistem
Precondition: Crew sedang membuat transaksi atau di menu pelanggan
Postcondition: Pelanggan baru berhasil ditambahkan

Flow Utama:
1. Crew klik "Tambah Pelanggan Baru"

   - Nama Pelanggan (text field)
   - Nomor HP (number field)

6. Sistem validasi input:
   - Nama tidak boleh kosong
   - HP harus format benar
   - Alamat tidak boleh kosong
7. Jika validasi gagal: Tampilkan error, highlight field error
8. Jika validasi sukses: Kirim ke server POST /v1/pelanggan/create
9. Server membuat record baru, return customer ID
10. Sistem simpan pelanggan ke cache lokal
11. Aplikasi kembali otomatis dan pre-select pelanggan baru ini

USE CASE 1.10 - MANAJEMEN GALON (PINJAM/KEMBALI)
---
Nama: Manajemen Galon (Pinjam & Kembali)
Aktor Utama: Crew
Sistem: Depo Air Mobile App
Tujuan: Crew mencatat galon yang dipinjam/dikembalikan pelanggan
Precondition: Crew sudah login
Postcondition: Mutasi galon tercatat di sistem

Flow Utama - PINJAM GALON:
1. Crew dari Dashboard klik "Manajemen Galon"
2. Aplikasi menampilkan halaman Galon:
   - Tab: Tersedia | Dipinjam | Rusak | Hilang
   - Tab default: "Tersedia"
3. Crew klik tab "Tersedia"
4. Menampilkan list galon dengan status "Tersedia"
5. Crew klik tombol "Pinjam"
6. Aplikasi buka form "Pinjam Galon":
   - Pilih Pelanggan (dropdown)
   - Pilih Galon (dari list tersedia)
   - Jumlah: auto 1, bisa edit
   - Catatan (opsional)
7. Crew pilih pelanggan & galon
8. Crew klik "Konfirmasi Pinjam"
9. Sistem update status galon: Tersedia → Dipinjam
10. Sistem catat mutasi: PINJAM
11. Sistem update customer: Total galon pinjam +1
12. Kirim ke server: POST /v1/galon/pinjam
13. Server simpan dan return response
14. Toast: "✓ Galon berhasil dipinjam"

Flow Utama - KEMBALIKAN GALON:
1. Crew dari halaman Galon, klik tab "Dipinjam"
2. Menampilkan galon yang sedang dipinjam
3. Crew klik 1 galon untuk detail
4. Detail menampilkan:
   - Kode galon
   - Customer yang pinjam
   - Tanggal pinjam
   - Kondisi saat dipinjam
   - Tombol: Kembalikan
5. Crew klik "Kembalikan"
6. Aplikasi buka form "Kembalikan Galon":
   - Display info galon & customer
   - Pilihan Kondisi: Baik, Rusak, Hilang
7. Crew pilih kondisi galon saat kembali
8. Crew klik "Konfirmasi Kembali"
9. Sistem update status galon sesuai kondisi:
   - Baik → Tersedia
   - Rusak → Rusak
   - Hilang → Hilang
10. Sistem catat mutasi: KEMBALI + kondisi
11. Sistem update customer: Total galon pinjam -1
12. Kirim ke server: POST /v1/galon/kembali
13. Server simpan dan return response
14. Toast: "✓ Galon berhasil dikembalikan"

Nama: Validasi Transaksi dari Crew
Aktor Utama: Manager
Sistem: Depo Air Mobile App
Tujuan: Manager menyetujui atau menolak transaksi yang dibuat crew
Precondition: Ada transaksi pending dari crew
Postcondition: Transaksi di-approve/reject, stok ter-update

Flow Utama - VIEW LIST PENDING:
1. Manager dari Dashboard klik "Validasi Transaksi"
2. Aplikasi fetch dari server: GET /v1/transaksi?status=pending
3. Aplikasi menampilkan list transaksi PENDING:
   - Nomor transaksi (TRX-xxx)
   - Nama crew yang buat
   - Nama customer
   - Total harga
   - Jam dibuat
   - Tombol: Detail
4. Manager bisa scroll lihat semua pending transaksi
5. Manager klik 1 transaksi untuk lihat detail

Flow Utama - DETAIL TRANSAKSI:
1. Aplikasi menampilkan halaman "Detail Transaksi" dengan:
   
   SECTION 1 - INFO UMUM:
   - Nomor Transaksi: TRX-20260523-001
   - Tanggal/Jam: 23/05/2026 09:30
   - Status: PENDING (MENUNGGU VALIDASI)
   - Crew: Budi Santoso
   
   SECTION 2 - DATA PELANGGAN:
   - Nama: Rudi Hermawan
   - Nomor HP: 0812345678
   - Alamat: Jl. Sudirman No.5
   - Total Pembelian: Rp 5.500.000 (from history)
   - Total Galon Pinjam: 3 galon
   
   SECTION 3 - DETAIL ITEM:
   ┌────────────────────────────────────────────┐
   │ Produk                   | Qty | Harga     │
   ├────────────────────────────────────────────┤
   │ Air Galon 19L            │ 5   │ 2.500.000 │
   │ Air Minum Kemasan 600ml  │ 10  │ 750.000   │
   │ Galon Baru (Pinjam)      │ 2   │ -         │
   │ Galon Lama (Kembali)     │ 1   │ -         │
   └────────────────────────────────────────────┘
   Total: Rp 3.250.000
   
   SECTION 4 - PEMBAYARAN:
   - Metode: QRIS
   - Status: ✓ Sudah Bayar
   - Nominal: Rp 3.250.000
   - Waktu Bayar: 23/05/2026 09:32
   - Reference ID: QRIS-12345678
   
   SECTION 5 - VALIDASI INFO:
   Manager bisa check:
   ✓ Stok produk cukup?
   ✓ Harga sesuai?
   ✓ Customer valid?
   ✓ Pembayaran valid?
   ✓ Galon status valid?
   
   SECTION 6 - AKSI:
   - Tombol BESAR: [APPROVE] [REJECT]

Flow Utama - APPROVE TRANSAKSI:
1. Manager review semua data
2. Jika semua OK, Manager klik "APPROVE"
3. Aplikasi minta konfirmasi: "Setuju validasi transaksi ini?"
4. Manager klik "Ya, Approve"
5. Sistem update status transaksi: PENDING → DIPROSES → SELESAI
6. Sistem kurangi stok produk:
   - Air Galon 19L: -5
   - Air Minum 600ml: -10
7. Sistem update status galon:
   - 2 Galon baru: Tersedia → Dipinjam (untuk customer ini)
   - 1 Galon lama: Dipinjam → Tersedia
8. Sistem update customer:
   - Total Galon Pinjam: +2 -1 = +1
   - Total Pembelian: +3.250.000
9. Kirim ke server: POST /v1/transaksi/validate (approve=true)
10. Server update database
11. Aplikasi tampilkan: "✓ Transaksi berhasil di-approve"
12. Kirim notifikasi ke crew: "Transaksi Anda di-approve"
13. Auto back ke list pending

Flow Utama - REJECT TRANSAKSI:
1. Manager review data
2. Jika ada masalah, Manager klik "REJECT"

  └─ Transaksi status: PENDING
  └─ Notifikasi ke Manager: Ada transaksi baru

T=0:30 - Manager: Lihat Widget
  └─ Manager: Melihat widget "5 transaksi pending"

T=0:35 - Manager: Validasi
  └─ Manager: Buka, review, approve/reject transaksi

T=0:40 - Sistem: Update
  └─ Jika APPROVE: Update stok, galon, customer, notif crew
  └─ Jika REJECT: Cancel transaksi, notif crew dengan alasan

T=0:45 - Crew: Notifikasi
  └─ Crew: Menerima notifikasi hasil validasi
  └─ Crew: Bisa lihat di riwayat transaksi


DATABASE RELATIONSHIPS:
---
Crew ─┬─→ Transaksi (1:Many)
      ├─→ Pelanggan (1:Many) [create/manage]
      └─→ Galon (1:Many) [pinjam/kembali record]

         ├─→ Produk [manage stok]
         ├─→ Crew [manage]
         └─→ Laporan [generate]
